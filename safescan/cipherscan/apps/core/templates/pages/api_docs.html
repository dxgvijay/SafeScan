Authentication

All API requests require authentication via an API key. Include your key in the Authorization header or as a query parameter.

API Key (Header)
Send your API key in the X-API-Key header
Bearer Token
Send your API key as a Bearer token in the Authorization header
curl
# With X-API-Key header
curl -H "X-API-Key: YOUR_API_KEY" \
  https://api.cipherscan.dev/v1/url-scan/

# With Bearer token
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.cipherscan.dev/v1/url-scan/

Get your API key from the API Settings page. Keep your key secret — never share it or expose it in client-side code.

Endpoints v1

The base URL for all API endpoints is:

base URL
https://api.cipherscan.dev/v1/

URL Scanner

POST/url-scan/

Submit a URL for security analysis. Returns a scan ID that can be used to retrieve results.

Request Body

ParameterTypeRequiredDescription
urlstringYesThe URL to scan (must be fully qualified)
callbackstringNoWebhook URL to notify when scan completes
GET/url-scan/{id}/

Retrieve the results of a URL scan by ID.

Path Parameters

ParameterTypeDescription
iduuidThe unique scan identifier returned from the POST request

File Scanner

POST/file-scan/

Upload a file for malware and threat analysis. Supports files up to 100MB.

Request (multipart/form-data)

ParameterTypeRequiredDescription
filefileYesThe file to scan
filenamestringNoOverride the filename for analysis

Phishing Analysis

POST/phishing/analyze/

Analyze email headers and content for phishing indicators.

ParameterTypeRequiredDescription
headersstringYesRaw email headers to analyze
bodystringNoEmail body content for deeper analysis

Network Tools

GET/dns-lookup/?domain={domain}&type={type}
GET/whois/?domain={domain}
GET/ssl-check/?host={host}&port={port}

Code Examples

Quickstart examples in popular programming languages.

Python
import requests

API_KEY = "YOUR_API_KEY"
url = "https://api.cipherscan.dev/v1/url-scan/"

headers = {"X-API-Key": API_KEY}
data = {"url": "https://example.com"}

response = requests.post(url, json=data, headers=headers)
scan_id = response.json()["id"]
print(f"Scan ID: {scan_id}")
JavaScript
const API_KEY = "YOUR_API_KEY";

const response = await fetch("https://api.cipherscan.dev/v1/url-scan/", {
  method: "POST",
  headers: {
    "X-API-Key": API_KEY,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({ url: "https://example.com" })
});

const data = await response.json();
console.log("Scan ID:", data.id);

Rate Limits

Rate limits are applied per API key. Exceeding these limits returns a 429 Too Many Requests response.

PlanRequests/MinRequests/HourRequests/Day
Free305001,000
Pro1203,60010,000
Enterprise60018,000Unlimited

Rate limit headers are included in every response: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset.

Error Codes

The API uses standard HTTP status codes. Errors return a JSON object with an error field describing the issue.

Status CodeDescription
200 OKRequest succeeded
400 Bad RequestInvalid request parameters
401 UnauthorizedMissing or invalid API key
403 ForbiddenAPI key lacks permissions for this endpoint
404 Not FoundResource not found
422 UnprocessableRequest validation failed
429 Too Many RequestsRate limit exceeded
500 Internal ErrorServer error — contact support if persistent
